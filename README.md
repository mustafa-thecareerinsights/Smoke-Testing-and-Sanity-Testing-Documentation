## Smoke Testing vs Sanity Testing

| Area | Smoke Testing | Sanity Testing |
|---|---|---|
| Purpose | Verifies whether the main build is stable enough for further testing | Verifies whether a specific change or fix works as expected |
| Scope | Broad and shallow | Narrow and focused |
| Timing | Performed after a new build is deployed | Performed after a bug fix, small enhancement, or targeted change |
| Goal | Confirm critical features are not broken | Confirm the changed area works and related functionality is stable |
